# feat — 功能规划

生成 `plan/feat/<branch-name>/PLAN.md`：

1. **功能目标** — 一句话描述这个功能解决什么问题
2. **技术选型** — 涉及的技术/库/外部服务
3. **目录结构** — 新增/修改的文件清单
4. **API 设计** — 新增接口（Method / Path / 功能）
5. **数据流** — 核心流程的数据流向
6. **核心参数** — 关键常量/配置项
7. **功能边界** — 做什么 / 不做什么
8. **实现顺序** — 基于依赖关系排列模块顺序

收尾：提示运行 `/task` 生成 TDD 任务清单（必须执行）
