# build — Solution Execute 参考

## 读取 TASK.md

- 构建配置任务。
- 脚本或包元数据任务。
- 构建命令验证任务。
- 产物验证任务。

## 执行顺序

1. 修改构建配置或元数据。
2. 运行构建或结构验证命令。
3. 验证预期产物，或记录为什么没有持久产物。
4. 更新 `TASK.md`.

## 验证

- 如有可用的构建命令或结构验证，必须记录。
- 产物验证检查路径、文件列表、元数据、manifest、大小、生成输出或下游加载行为。
- 避免把旧产物误当成本次新输出。

## TASK.md 更新

- 只有在构建和产物验证通过，或已记录无产物原因与可观察输出后，才能把 build 任务标记为 `[x]`。

## 停止并进入 review

如果构建无法验证、产物预期不清楚，或配置变更超出 `SOLUTION.md` 范围，停止并进入 review。
