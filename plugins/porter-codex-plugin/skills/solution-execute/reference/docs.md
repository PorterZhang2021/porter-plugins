# docs — Solution Execute 参考

## 读取 TASK.md

- 文档编写任务。
- 结构、链接、术语和示例验证任务。

## 执行顺序

1. 更新指定文档文件。
2. 验证标题、链接、示例、术语和引用路径。
3. 更新 `TASK.md`.

## 验证

- 纯文档工作可以使用结构审查、链接/路径检查、Markdown 围栏检查或 diff 审查验证。
- 示例如果可执行或对路径敏感，需要检查。

## TASK.md 更新

- 指定内容存在且验证证据已记录后，才能把 docs 任务标记为 `[x]`。
- 停止前记录缺失的素材或未解决的措辞决策。

## 停止并进入 review

如果文档揭示缺失的产品行为、与 `SOLUTION.md` 矛盾，或需要范围变更，停止并进入 review。
