# perf — Solution Execute 参考

## 读取 TASK.md

- 基线度量或采集计划任务。
- 瓶颈确认任务。
- 优化实现任务。
- 优化后验证任务。

## 执行顺序

1. 采集或确认基线数据。
2. 确认瓶颈。
3. 只有基线和瓶颈仍支持该方向时，才实现优化。
4. 优化后再次度量。
5. 与基线对比。
6. 更新 `TASK.md`.

## 验证

- 有可用基线时，必须记录命令、输入、环境和结果。
- 优化必须保持已接受行为不变。
- 优化后度量必须与基线对比。
- 如果无法形成持久指标，记录限制和可观察证据。

## TASK.md 更新

- 变更后度量或限制记录完成前，不得标记优化完成。
- 记录基线、对比结果和任何变化的瓶颈假设。

## 停止并进入 review

如果基线数据与预期瓶颈矛盾、优化方向过期，或已接受的性能标准需要变化，停止并进入 review。
