# docs — Solution Task 参考

## 读取 SOLUTION.md

- `目标`
- `范围`
- `类型专项分析`
- `拟议变更`
- `验收标准`

## 任务类型

- 起草或更新文档。
- 检查结构、链接和术语。
- 存在示例时验证示例。

## 顺序

- 先写内容任务，再写审查任务。
- 仅文档变更使用"无业务逻辑，无需测试；通过结构审查验证"。

## 模板

```markdown
## Task N：<文档章节>

无业务逻辑，无需测试；通过结构审查验证。

- [ ] 更新 `<doc_path>`
- [ ] 验收标准：<SOLUTION.md 验收标准中预期的文档结果>
- [ ] 验证方式：标题、链接、示例、术语和引用路径已检查
```
