# chore — Solution 模板

## 类型专项分析必填字段

1. **变更内容** — 具体维护事项，例如调整配置、清理文件、整理元数据。
2. **变更原因** — 为什么需要维护。
3. **影响范围** — 可能影响哪些流程、文件或用户习惯。
4. **风险评估** — 可能出错的地方。
5. **回滚方案** — 出问题如何恢复。
6. **验证方式** — 如何确认维护变更生效。

## 视觉模型

`chore` 通常不强制 Mermaid。

- 如果维护项影响 workflow、目录迁移或多文件关系，建议使用 `flowchart`。
- 小范围元数据或文案调整可以写明"无图，原因：..."。

## 验收标准写法

- 维护目标完成。
- 影响范围清楚。
- 有回滚或恢复思路。
- 验证方式可执行。

## 待确认建议

- 维护范围是否正确。
- 风险评估是否完整。
- 回滚方案是否可接受。
- 是否需要拆成更小的维护项。

## solution-task 提示

- 视复杂度决定是否需要测试。
- 至少需要结构审查或命令验证。
