# docs — Solution 模板

## 类型专项分析必填字段

1. **文档目标** — 面向谁，解决什么信息缺口。
2. **目标读者** — 使用者、维护者、贡献者或当前项目协作者。
3. **文档结构** — 章节大纲。
4. **内容来源** — 参考哪些代码、文档、配置或设计。
5. **更新范围** — 新增或修改哪些文档文件。
6. **验收方式** — 如何确认信息完整、路径有效、术语一致。

## 视觉模型

`docs` 可选 Mermaid 图。

- 面向流程说明时，使用 `flowchart`。
- 面向架构或信息结构说明时，使用 `flowchart` 或 `mindmap`。
- 纯文字说明或小范围文档修订可以写明"无图，原因：..."。

## 验收标准写法

- 目标读者能根据文档完成对应动作或理解对应规则。
- 引用路径有效。
- 不引入与现有说明冲突的描述。

## 待确认建议

- 目标读者是否正确。
- 文档结构是否符合预期。
- 内容来源是否完整。
- 是否有必须避免的表述或术语。

## solution-task 提示

- 生成写作任务。
- 通常无需自动化测试，通过结构审查和链接检查验证。
