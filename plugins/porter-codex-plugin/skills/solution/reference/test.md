# test — Solution 模板

## 类型专项分析必填字段

1. **测试目标** — 补充哪个模块、流程或场景的测试。
2. **覆盖缺口** — 当前缺少哪些正常路径、边界条件或异常路径。
3. **测试类型** — 单元测试 / 集成测试 / 端到端测试 / 手动结构验证。
4. **用例设计** — Given / When / Then 或等价描述。
5. **Mock 策略** — 哪些外部依赖需要 Mock；没有则写"无"。
6. **业务行为变更** — 是否改业务行为；通常应为"不改"。

## 视觉模型

`test` 可选 Mermaid 图。

- 测试覆盖一个流程时，使用 `flowchart` 表达被测路径。
- 测试多个组件交互时，使用 `sequenceDiagram` 表达调用关系。
- 单纯补充局部单元测试时可以写明"无图，原因：..."。

## 验收标准写法

- 覆盖缺口被测试覆盖。
- 测试能在项目现有方式下运行或检查。
- 不引入非预期业务行为变更。

## 待确认建议

- 覆盖目标是否正确。
- 测试类型是否合适。
- 用例设计是否覆盖关键路径。
- 是否确认不改业务行为。

## solution-task 提示

- 只生成测试任务，不生成实现任务，除非 solution 明确允许补测试辅助代码。
