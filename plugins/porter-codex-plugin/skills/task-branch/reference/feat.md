# feat — 功能任务清单格式

逐模块与用户确认，每个模块使用以下格式：

**有业务逻辑的模块：**

```markdown
## Task N：<模块名>

- [ ] **[测试]** `<测试文件路径>`（先查看项目已有测试文件结构，遵循已有约定；若无现有测试，询问用户偏好）
  - **用例：<用例标题，描述业务行为>**
    - Given：<前置条件，准备什么数据或 Mock 什么行为>
    - When：<触发什么操作>
    - Then：<验证结果，断言什么值 / 调用次数 / 状态变化>
  - ...
- [ ] **[实现]** `<module_path>`（测试全部红色后开始）
  - `<函数名>(<参数>)` — 按项目语言写签名
  - ...
```

**无业务逻辑的模块（常量、配置）：**

```markdown
## Task N：<模块名>

无业务逻辑，无需测试。

- [ ] 创建 `<file_path>`
- [ ] 定义 ...
```
