# fix — Bug 修复任务清单格式

**第一个任务必须是能复现 Bug 的失败测试：**

```markdown
## Task 1：复现测试（必须先完成）

- [ ] **[测试]** `<测试文件路径>`（先查看项目已有测试文件结构，遵循已有约定；若无，询问用户偏好）
  - **用例：<复现 Bug 的场景描述>**
    - Given：<触发 Bug 的前置条件>
    - When：<触发操作>
    - Then：<期望的正确行为（当前会失败）>
- [ ] **[修复]** `<file_path>`（复现测试红色后开始）
  - 具体修改点

## Task 2：回归测试补充（可选）

- [ ] **用例：<边界场景描述>**
    - Given / When / Then
```
