# CLAUDE.md Workshop

帮助用户为新项目结对编写 CLAUDE.md 操作手册。

## 目标

基于固定的四节结构，逐节生成草稿，用户微调确认，最终生成项目根目录下的 `CLAUDE.md`。

## 前置条件

在开始任何内容生成前，先检查 `.claude/constitution.md` 是否存在：

- **存在** → 继续执行后续流程
- **不存在** → 暂停，告知用户"未找到项目宪法，正在启动 `/constitution` 流程"，然后立即执行 `/constitution` 命令的完整流程，宪法完成后再继续 CLAUDE.md 的编写

## 结构模板

收集完所有节所需信息后，一次性生成完整 CLAUDE.md 草稿，展示全文，写入文件，最后询问：**"CLAUDE.md 已写入，是否运行 `/commit` 提交？"**

各节所需输入如下（统一在开始前询问）：

### 节 1：核心原则导入（最高优先级）

固定内容，无需讨论，直接生成：

```markdown
## #---核心原则导入(最高优先级)-----#

明确导入项目宪法，确保 AI 在思考任何问题前，都已加载核心原则。

@.claude/constitution.md
```

### 节 2：核心使命与角色设定

询问用户：
- Claude 扮演什么角色？（资深工程师 / 结对编程伙伴 / 严格执行者）
- 项目使命一句话概括是什么？

基于回答生成草稿，格式：
```markdown
## #---核心使命与角色设定-----#

**角色：** ...
**使命：** ...
**行为准则：**
- ...
```

### 节 3：技术栈与环境

询问用户：
- 主要技术栈（语言、框架、数据库、外部服务）？
- 启动命令是什么？
- 有哪些必要的环境变量？

基于回答生成草稿，格式：
```markdown
## #---技术栈与环境-----#

**技术栈：** ...
**环境初始化：** ...
**环境变量：** ...
```

### 节 4：Git 与版本控制

询问用户是否有特殊的分支或提交规范，否则使用默认规范（`/new-branch` + `/commit`）生成草稿。

### 节 5：AI 协作指令

询问用户：
- 有哪些明确禁止的行为？
- 工作流程是否强制 TDD？
- 代码审查有哪些硬性标准？

**明确禁止**草稿须包含以下固定默认项（不可删除），再追加用户自定义项：

```markdown
**明确禁止：**
- 在 worktree 中操作时，将文件写入主仓库路径（须先 `pwd` 确认当前目录，所有写入路径必须以 worktree 目录为基准）
- （用户自定义禁止项）
```

工作流程草稿须包含以下固定步骤（可根据用户回答调整措辞）：

```markdown
**工作流程：**

1. 收到需求 → 理解并复述，确认无歧义
2. 写 `TASK.md` → 拆分子任务，逐条列出，等待用户确认
3. 先写测试 → 按宪法第二条（Red → Green → Refactor），等待用户确认测试用例
4. 写实现 → 最小可通过测试的代码
5. 运行测试 → 确认全部通过，更新 `TASK.md` 对应条目为完成
6. 使用 `/commit` 提交
```

## 收尾

- 写入 `CLAUDE.md`
- 询问：**"CLAUDE.md 已写入，是否运行 `/setup-permissions` 配置项目权限？"**

## 注意事项

- CLAUDE.md 是操作手册，不是原则文档——写具体的命令、工具、工作流程
- 节与节之间保持统一的标题格式：`## #---节名-----#`
- 每节生成后直接继续，无需用户逐节确认
